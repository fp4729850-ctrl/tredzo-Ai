# Requirements Document

## 1. Application Overview

**Application Name**: Advanced Auto Trading Bot System

**Description**: An AI-based cryptocurrency trading bot web application that automatically scans market Top Gainers and Top Losers across multiple timeframes to identify high-probability trade opportunities. Users can integrate their own PineScript strategies, connect their Binance API keys for automated trading execution, and backtest strategies against real historical data from Binance.

## 2. Users and Use Scenarios

**Target Users**: Cryptocurrency traders who want to automate their trading strategies using custom PineScript logic and Binance API integration.

**Core Use Scenarios**:
- Users register and log in to access the trading bot system
- Users add their Binance API credentials to enable automated trading
- Users upload or paste their PineScript trading strategies
- Users backtest their strategies against real historical market data from Binance to validate performance
- System scans market for Top Gainers and Top Losers to generate trade signals
- Bot executes trades automatically based on user's PineScript strategy and AI analysis
- Users monitor active positions, trade history, and performance metrics

## 4. Business Rules and Logic

### 4.1 Market Scanning Logic
- System continuously scans cryptocurrency market data in real-time
- Identifies Top Gainers (coins with highest price increases) and Top Losers (coins with highest price decreases)
- For Top Gainers: AI analyzes market momentum and volume to detect Strong Sell Opportunities
- For Top Losers: AI analyzes trend reversal patterns to detect Strong Buy Opportunities
- Multi-timeframe analysis is performed to validate signals across different time periods

### 4.2 PineScript Strategy Execution
- AI agent parses user's PineScript strategy code to extract trading rules
- System monitors market conditions that match the strategy's entry/exit criteria
- When strategy conditions are met and AI confidence score is sufficient, system generates trade signal
- Trade signal includes coin symbol, action (buy/sell), suggested entry price, stop loss, and take profit levels

### 4.3 Backtesting Execution Logic
- User selects a saved PineScript strategy, coin symbol, date range, and timeframe
- Edge Function fetches real historical OHLCV data from Binance public klines API endpoint: GET https://api.binance.com/api/v3/klines
- Request parameters include: symbol, interval (mapped from user-selected timeframe), startTime (converted to Unix timestamp in milliseconds), endTime (converted to Unix timestamp in milliseconds), limit (max 1000 per request)
- Since a date range can span more than 1000 bars, Edge Function paginates multiple requests:
  - Calculate total number of bars needed based on date range and timeframe
  - Make initial request with limit=1000
  - Use the last candle's timestamp from previous response as startTime for next request
  - Repeat until endTime is reached or all data is retrieved
  - Aggregate all paginated responses into complete OHLCV dataset
- Edge Function simulates the PineScript strategy logic against real historical data bar by bar
- For each bar, system evaluates strategy entry and exit conditions
- When entry signal is generated, system simulates opening a position at the bar's close price
- When exit signal is generated (stop loss, take profit, or strategy exit condition), system simulates closing the position
- System calculates P&L for each simulated trade
- System aggregates results to generate equity curve, trade list, and performance metrics
- Edge Function generates ohlcv_sample field containing downsampled version of fetched OHLCV data (max 500 candles) to keep response payload manageable
- Results including ohlcv_sample are returned to frontend and displayed on Backtesting Page
- User can save backtest results with a custom name for future reference
- No Binance API key is required for accessing public market data endpoints

### 4.4 Candlestick Chart Display Logic
- Frontend receives ohlcv_sample data from Edge Function response
- Candlestick chart component renders OHLC candles using lightweight-charts library
- Trade entry markers are positioned at corresponding candle timestamps: green markers for BUY entries, red markers for SELL entries
- Trade exit markers are positioned at corresponding candle timestamps: green markers for profitable exits, red markers for loss exits
- User hovers over candle to view tooltip with OHLCV data
- User hovers over trade marker to view tooltip with entry/exit trade details (symbol, action, price, timestamp, P&L)

### 4.5 Automated Trading Flow
- When trade signal is generated and bot is enabled, system uses user's Binance API credentials to execute trade
- System places market or limit orders on Binance based on signal parameters
- Open positions are tracked in real-time
- When exit conditions are met (stop loss, take profit, or strategy exit signal), system automatically closes position
- All trade executions are logged in Trade History

### 4.6 Multi-User Isolation
- Each user has independent Binance API credentials
- Each user can configure their own PineScript strategies
- Trading execution and position tracking are isolated per user
- Users cannot access other users' data or trading activities
- Backtest results are isolated per user

### 4.7 Signal Confidence Scoring
- AI analyzes multiple factors: market momentum, volume, trend strength, PineScript strategy alignment
- Generates confidence score (0-100%) for each trade signal
- Users can configure minimum confidence threshold for automated execution

## 5. Exception and Boundary Cases

| Scenario | Handling |
|----------|----------|
| User inputs invalid Binance API credentials | Display error message, prevent bot activation until valid credentials are provided |
| PineScript code contains syntax errors | Display error message with details, prevent strategy activation |
| Binance API rate limit exceeded | Pause trading operations temporarily, display notification to user |
| Insufficient balance in Binance account | Skip trade execution, log event in Trade History with reason |
| Network connection lost during trade execution | Retry operation, if fails after retries, log error and notify user |
| Market data feed interruption | Pause signal generation, display warning on Dashboard |
| User attempts to delete active strategy with open positions | Display confirmation dialog warning about open positions |
| Stop loss or take profit triggered | Execute exit order immediately, update position status |
| User selects invalid date range for backtest (end date before start date) | Display error message, prevent backtest execution |
| Historical data not available for selected symbol or date range | Display error message indicating data unavailability |
| Backtest execution takes too long | Display loading indicator, allow user to cancel backtest |
| User attempts to run backtest without selecting a strategy | Display error message prompting user to select a strategy |
| Binance klines API is unreachable during backtest | Display clear error message: Unable to fetch historical data from Binance. Please check your network connection and try again. |
| Binance klines API returns error response during backtest | Display error message with details from Binance API response, prevent backtest execution |
| Pagination fails during multi-request data fetching | Display error message indicating incomplete data retrieval, prevent backtest execution |
| ohlcv_sample data exceeds 500 candles | Edge Function downsamples data to max 500 candles before returning response |
| Candlestick chart fails to render | Display error message, allow user to view other backtest results (equity curve, trade list, metrics) |
| Trade markers do not align with candles due to timestamp mismatch | System matches trade timestamps to nearest available candle timestamp |
